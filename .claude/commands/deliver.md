# /deliver — Fullstack → PM handoff

Ты — оркестратор. Запускаешь цепочку: fullstack-агент реализует задачу, затем pm-агент закрывает её в Notion.

Аргумент команды (`$ARGUMENTS`) — описание задачи или ссылка на Notion task.

## Шаг 1 — Реализация (fullstack-агент)

Вызови sub-agent `fullstack` со следующим промптом:

> Задача: $ARGUMENTS
>
> Реализуй задачу согласно своему workflow. По завершении верни структурированный отчёт:
> - Task name
> - Notion task URL (если был передан)
> - PR URL
> - Что сделано (1–3 строки)
> - Acceptance criteria — все покрыты? (да / нет + что не покрыто)
> - Deploy проверен? (да / нет)

Дождись ответа fullstack-агента.

## Шаг 2 — Закрытие (pm-агент)

Если fullstack-агент вернул `Acceptance criteria: да` и `Deploy проверен: да` — вызови sub-agent `pm` со следующим промптом:

> Fullstack-агент завершил задачу. Вот отчёт:
> <вставь отчёт из Шага 1>
>
> Твои действия:
> 1. Переведи задачу в Notion в статус Done.
> 2. Убедись, что в PR description есть ссылка на Notion task.
> 3. Если задача была частью спринта — обнови прогресс спринта.
> 4. Проверь: не появились ли новые задачи из отчёта fullstack-агента, которые нужно завести.

Если acceptance criteria не все покрыты или deploy не проверен — **не вызывай pm-агента**. Сообщи Alisa что блокирует сдачу задачи, и верни задачу fullstack-агенту.

## Итог

После завершения обоих шагов выведи короткое резюме:
- Задача: <название>
- PR: <ссылка>
- Notion: <ссылка>
- Статус: Done ✓
